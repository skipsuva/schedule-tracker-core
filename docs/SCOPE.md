# Product Scope

This document defines **what Keel is and is not**, per version.

---

## V1 Scope (Must Ship)

### Core Capabilities
- User authentication (email + Google)
- Household creation on first login
- Household membership
- Children management
- Activities (templates)
- Activity instances (dated events)
- Basic calendar view
- Cost tracking tied to events or standalone
- Read-only reporting (monthly / upcoming)

### Constraints
- One household per user
- No invitations
- No recurring rules
- No notifications
- No budgets

---

## Explicitly Out of Scope (V1)

- Mobile apps
- Calendar sync (Google / Apple)
- Reminders / notifications
- Recurring events
- Advanced permissions
- Payments

---

## V2 (Planned / Likely)

- Recurring activity rules
- Household invitations
- Multi-household support
- Cost summaries & budgets
- Seasonal planning (camps)
- Read-only caregiver access
- Mobile apps (shared logic)
- AI queries (“what’s next week?”)

---

## Anti-Goals

- Becoming a full calendar replacement
- High-frequency notifications
- Over-automation

Keel is meant to be **structural, not noisy**.
